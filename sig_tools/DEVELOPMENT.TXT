Mortar Development guide
----------------------

Last update: 8/22/2012

Code Style Guidelines:
---------------------

- Filenames should be all lowercase. Use underscores if necessary to separate words.
- Function names should be in lowercase with underscores. 
- Variable names can be in lowercase with underscores or mixed case beginning with lowercase.
  Example: sigma, max_value, maxValue

- Constants should be all uppercase
  Example: MAX_ITERATIONS, DEFAULT_COLOR

- Use spaces following commas and around equal signs to improve readability
  Example: 
  	   col_collapser(ds, args.col_fields, hfun) 
	   ds.mat = mat;

- Avoid overriding inbuilt functions and variables
  Example: Use ii and jj instead of i and j for iterators

- Prefer using column major order for numeric arrays
  Example: indices = [1; 2; 3]

- Observe the 75 column limit for each line. Use the '...' operator to continue statements.
  Example:
	[~, gp_tmp, ~] = get_groupvar(ds.rdesc, ds.rhd,...
        [row_fields, ds.rhd{i}]);

	% Long strings can be broken up using the concatenation operator
	s = ['This is a really long string that wont',...
	'fit on one line']

- Comments should go  on a separate line. Avoid end of line comments.

- For cell arrays, especially parameter names use a max of three parameters per line
  Example:
	pnames = {'row_fields', 'col_fields', 'fun',...
               'dim'};

- Use Matlab style for help documentaion.
  Example:
  DISP Display array.
    DISP(X) displays the array, without printing the array name.  In
    all other ways it's the same as leaving the semicolon off an
    expression except that empty arrays don't display.
 
    If X is a string, the text is displayed.
 
    See also int2str, num2str, sprintf, rats, format.


- Compiled scripts should be named: XXX_tool.m
